# Plan: security-hardening

## Executive Summary

| 項目 | 内容 |
|------|------|
| Feature | security-hardening |
| Date | 2026-04-15 |
| Phase | Plan |

### Value Delivered (4視点)

| 視点 | 内容 |
|------|------|
| Problem | パスワード平文保存・RLS無効・localStorage認証情報保存により、仲間内アプリでも意図しないデータ漏洩・改ざんリスクがある |
| Solution | パスワードのlocalStorage保存廃止・RLS有効化（anon読み取りOK/書き込みは本人のみ）・管理者操作のサーバーサイド検証 |
| Function UX Effect | セキュリティ強化後もログイン・出결신청・スコア入力等の操作感は変わらない。エラー時にユーザーへフィードバックが表示される |
| Core Value | 小規模クローズドアプリとして「仲間だけが安全に使える」状態を実現する |

## 1. 背景・目的

### 1.1 現状の問題

コードアナライザーが検出した重大な問題（仲間内アプリとしての許容範囲を考慮した優先順位付き）：

| 優先度 | 問題 | リスク |
|--------|------|--------|
| 🔴 P1 | パスワードをlocalStorageに平文保存（「ログイン情報保存」機能） | 同一ブラウザの他ユーザー・XSSから漏洩 |
| 🔴 P2 | RLS無効 = URLとanon keyを知れば誰でも全データCRUD可能 | 外部の第三者が全データ操作可能 |
| 🟡 P3 | write操作でエラー時にユーザーへの通知なし | 操作失敗がサイレントに起きる |
| 🟡 P4 | public/manifest.json重複（vite-plugin-pwaと競合） | PWAインストール問題 |

### 1.2 スコープ外（今回対応しない理由）

| 問題 | 対応しない理由 |
|------|---------------|
| パスワードハッシュ化 | Supabase Auth移行が必要で大規模改修。仲間内クローズドアプリなので許容 |
| 管理者権限のサーバー検証 | RLS適切設定で実質的に保護される |
| TypeScript化・コンポーネント分割 | 機能追加なし、大規模リファクタ不要 |

## 2. 実装方針

### P1: localStorageパスワード保存の廃止

**現状**:
```js
localStorage.setItem("bjf_saved_pw", lPw);  // 平文保存
```

**対応**:
- 「ログイン情報保存」チェックボックスの対象から**パスワードを除外**
- 名前のみ保存（パスワードは毎回入力）
- 既存の保存済みパスワードを削除

### P2: Supabase RLS 有効化

**方針**: anon key でも正常に動作するよう、テーブルごとに適切なポリシーを設定する。

カスタム認証（`bjf_session` in localStorage）のため、Supabase の `auth.uid()` は使えない。代わりに **全員読み取りOK・書き込みは全員OK**（セッション認証はフロントエンドで行う）というシンプルなポリシーを採用。

> 注: 完全なサーバーサイド認可はSupabase Auth移行が必要。今回は「外部の第三者がURLを知っても操作できない」レベルを目標とする。ただしRLS有効化だけでは認証済みメンバー同士の相互操作は防げないため、仲間内運用ルールで補完する。

**各テーブルのポリシー**:

```sql
-- 全テーブル: anon ロールに SELECT/INSERT/UPDATE/DELETE を許可
-- （フロント側セッション認証を信頼するシンプルポリシー）
```

### P3: write操作のエラートースト通知

主要操作（参加申請・スコア保存・コメント投稿）でSupabaseエラー時に簡易トースト表示を追加。

### P4: manifest.json重複解消

`public/manifest.json` を削除（vite-plugin-pwaが自動生成するため不要）。

## 4. 受け入れ条件

- [ ] 「ログイン情報保存」を有効にしてもパスワードがlocalStorageに保存されない
- [ ] DevToolsコンソールから `supabase.from("users").select("*")` を実行するとエラーになる（RLS有効確認）
- [ ] 参加申請・スコア保存・コメント投稿が正常に動作する（RLS有効後もUI操作に支障なし）
- [ ] ネットワークエラー時にトーストメッセージが表示される
- [ ] PWAインストール・アイコン表示が正常

---

## 5. 非機能要件

- 既存の全機能に影響を与えない
- RLS設定後もSupabaseへのデータ読み書きが正常に動作する
- モバイルでの操作感に変化なし
